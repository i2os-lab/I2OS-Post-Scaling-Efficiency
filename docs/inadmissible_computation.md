# Inadmissible Computation

## Definition

Inadmissible Computation is computation spent on trajectories that cannot preserve admissible continuity.

In simpler terms:

```text
Inadmissible Computation is computation spent on futures that cannot stably continue.
```

---

## Examples

- hallucinated generation
- unsupported reasoning paths
- unsafe tool execution
- unrecoverable state transitions
- regeneration loops
- human-unverifiable complexity
- agentic drift
- future-incompatible decisions

---

## Why It Matters

Inadmissible Computation increases:

- inference cost
- regeneration cost
- correction cost
- human review burden
- rollback cost
- trust repair cost
- external action risk

I2OS reduces this by excluding inadmissible futures before generation or execution.

---

## Core Statement

The next phase of AI efficiency is not only faster computation.

It is the reduction of computation that should never have been generated.
